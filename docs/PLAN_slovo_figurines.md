# План: статуэтки + вводные Слову + карты (M/G)

Удобный срез **ближайшей** фичи по арке Слова. Не вся арка — только то, что нужно, чтобы:

1. Слово в диалоге говорил по новым вводным (коллекция, миелофон, миссия — без обещаний неготового).
2. Статуэтки были **предсказуемо расставлены** и видны: **галактика → система**, **система/локатор → точка**.
3. **M** и **G** открывали нужную карту и **в полёте, и у причала**.

Канон арки: `docs/GUIDE.md` → «Сюжетная арка: бог Слово».  
Сейчас: 1–2 статуэтки **на систему**, локальный сид; на галактической карте их нет; M/G в доке молчат.

---

## Цель MVP (этот план)

| Что | Готово когда |
|-----|----------------|
| Карты | G — галактика с маркерами систем, где есть живая статуэтка; M — системная карта + локатор с точной позицией/именем |
| Клавиши | M → вкладка system, G → galaxy; **и docked, и flight**; повтор своей — закрыть (в доке — уйти с карты на planet/station) |
| Расстановка | Пока **клиентский** детерминизм (общий сид галактики), но API такой, что сервер потом подменит источник без смены UI |
| Слово | Промпт + guide: коллекционер, миелофон, миссия «собери»; не обещает «Святой Дух»/полку/одушевление как готовое |
| Экономика | `figurine` basePrice → 0; в торге/промпте — не продавать за кредиты |

Вне MVP (следующие куски): «Святой Дух» в трафике, ×10/×100, щит-блик, миссия-дар миелофона,
1 на галактику сервером, полка Креста; **одолжить «Святой Дух»** при хорошем stance — на борту
один слот с переключаемыми режимами (скуп / оживитель / пожиратель / клонировщик).

## Срез 1 — Данные: «где статуэтки в галактике»

**Проблема:** `world.figurines` живёт только в **текущей** системе. Галактическая карта не знает чужие системы.

**Сделать (домен, без сервера пока):**

1. Функция уровня галактики, детерминированная от `galaxySeed` (+ epoch/delta при правках бога):
   ```ts
   // например packages/sim/src/domain/galaxy/figurineSites.ts
   type FigurineSite = {
     systemIndex: number
     titleId: string
     // опционально: seed для орбиты внутри системы
   }
   figurineSitesInGalaxy(seed, delta?): FigurineSite[]
   ```
2. Правило MVP (пока не «1 на всю галактику»):
   - либо **подмножество систем** (например ~N штук на галактику из сида — ближе к канону «редко»);
   - либо временно: маркер на галкарте для **текущей** + явный список из `figurineSitesInGalaxy` с COUNT на галактику.
   - **Рекомендация:** `FIGURINE.GALAXY_COUNT` (например 8–20) уникальных `systemIndex` из сида; в системе при `enterSystem`/`placeFigurines` — **если система в списке**, ставить 1 статуэтку с заданным `titleId` (не 1–2 random).
3. Контракт под сервер: тот же тип `FigurineSite[]`; позже источник = ответ сервера, клиентский генератор — fallback/offline.

**Файлы:** `config/figurines.ts`, новый `galaxy/figurineSites.ts`, `world/figurines.ts` (`placeFigurines` читает site для `world.systemIndex`), тесты на детерминизм и «одна на систему из списка».

**Приёмка:** два `createWorld` с одним сидом → одни и те же системы с статуэтками; система не из списка → `figurines.length === 0`.

## Сервер (закладка, не блокер MVP)

| Сейчас (MVP) | Потом |
|--------------|--------|
| `figurineSitesInGalaxy(seed)` на клиенте | Сервер шлёт тот же JSON / сидит в authoritative world |
| Подбор локальный | RPC: scoop → сервер подтверждает, сайт пропадает у всех |
| Полкa Креста нет | Серверное хранилище коллекции Слова |

UI карт не должен знать, откуда массив sites — только `sites: FigurineSite[]` из мира/контекста.

## Риски

- **Слишком много маркеров на G**, если оставить 1–2 на каждую из 2500 систем — карта умрёт. Поэтому срез 1 обязан сузить до `GALAXY_COUNT`.
- **Правки бога (delta)** сдвигают системы — sites пересчитывать от seed+delta или якорить по стабильному id системы.
- **Док + M/G:** не закрывать всю консоль у причала — только смена вкладки.
