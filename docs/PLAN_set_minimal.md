# Минимальная сеть — первый прогон (Supabase-first, 2026-07-12)

Цель: зайти с двух браузеров, увидеть друг друга в одной системе. Всё лишнее — потом.

## Стек: только Supabase, своего сервера НЕТ

Supabase закрывает весь минимум; Colyseus добавим отдельной фазой, когда дойдём до реального
боя между игроками (авторитетный сервер тактов). Для присутствия он не нужен.

- **Auth** — Supabase Auth (email+password). Хеширование, TLS, сессии — на их стороне, руками
  аутентификацию НЕ пишем (там легче всего накосячить). Браузерный менеджер паролей работает
  как обычно (форма с `autocomplete`). Сессия (JWT) хранится supabase-js сама, обновляется автоматически.
- **Сейвы** — таблица Postgres `saves` (`user_id` → `PlayerSave` JSON), тот же `serializePlayer`.
  Источник правды — сервер. **RLS**: каждый читает/пишет ТОЛЬКО свою строку (`auth.uid() = user_id`).
- **Присутствие** — Supabase Realtime: канал на систему (`system:<seed>:<index>`), Presence = «кто тут»,
  Broadcast = позиция/ориентация ~10–20 Гц. Домен ЧИСТ: удалённые игроки живут в app/render (кинематические
  метки, интерполяция), в `packages/sim` сеть не просачивается.

## Что поднимает пользователь

1. Завести бесплатный проект на supabase.com.
2. Прислать **Project URL** и **anon public key** (Settings → API) → в `apps/web/.env.local`:
   `VITE_SUPABASE_URL=…`, `VITE_SUPABASE_ANON_KEY=…` (anon-ключ безопасен в клиенте — доступ рулит RLS).
3. Один раз выполнить `docs/supabase.sql` в Supabase SQL Editor (таблица + RLS + политики).
4. Больше ничего не хостить: веб как запускался `npm run dev`, так и запускается.

## Клиент (apps/web)

- `app/net/supabase.ts` — клиент из env. Нет env → **офлайн-режим** (текущий localStorage-поток),
  чтобы игра не умирала без ключей и master-ветка не ломалась. Есть env → онлайн (логин + серверные сейвы + presence).
- Экран **логина/регистрации** (email+password, `autocomplete`) — гейт перед игрой в онлайн-режиме.
- **Сейвы**: при входе тянем из `saves`; автосейв по стыковке пишет в `saves` (upsert по `user_id`).
- **Presence**: подписка на канал системы; чужие игроки — маркеры на радаре/карте и простой меш в мире.
  Уведомление «кто-то вошёл» + галочка «следить» — следом за базовыми маркерами.

## Безопасность (держим по умолчанию)

- Аутентификацию не пишем сами — Supabase Auth.
- RLS включён на `saves`; без политики строки не видны никому. Проверяем, что чужую строку не прочитать.
- anon-ключ публичен намеренно; секрет (service_role) в клиент НЕ кладём никогда.
- HTTPS — на стороне Supabase (их домен) и хостинга веба; локально хватает `localhost`.

## Границы (сейчас НЕ делаем)

- Ни боя между игроками, ни коллизий, ни синхронного урона — только присутствие и метки.
- Colyseus, авторитетный тик-сервер — позже.
- Карьера профессий, звания, цели — отдельная фаза.
