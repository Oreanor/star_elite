# Реалтайм-сеть: встреча в системе + кооп-бой (простой путь, 2026-07-12)

Консолидирует решения обсуждения. Уточняет `PLAN_set_minimal.md` (тот описывал только
присутствие; здесь — движущиеся борта в мире и бой). Реализация идёт на **Firebase**,
а не Supabase/Colyseus, как в старых `TZ_*`/`PLAN_setevoy_komponent.md`.

## Что уже есть (не переделываем)

- **Auth** — Firebase Auth (email+пароль), `app/net/account.ts`. Офлайн без ключей.
- **Сейвы** — Firestore `saves/{uid}`, `PlayerSave` JSON, автосейв при стыковке.
- **Presence (медленный, метаданные)** — RTDB `presence/{uid}` раз в 2 с: имя, systemIndex,
  станция, вид/лицо/профессия, paused, **абсолютная поза x/y/z**. `onDisconnect` чистит.
  Публикует `PresencePublisher` (`app/App.tsx`).
- **Чат игрок↔игрок** — RTDB `chats/{cid}` + инбокс-вызовы (`app/net/chat.ts`, `ui/chat/`).
- **Список «В СЕТИ»** — текст в Консоли (`ui/console/Console.tsx`).

## Ключевые решения

1. **Простой путь: только Firebase RTDB, Colyseus НЕ поднимаем.** Каждый клиент авторитетен
   над СВОИМ HP (кооп с друзьями ≤10, читерно-но-неважно; побег-перезагрузкой уже разрешён).
   Colyseus/lag-comp отложены — домен чистый + шов `RemoteController` оставляем, чтобы
   перейти позже без переписывания.

2. **Два РАЗНЫХ presence, не путать** (разведены назначением и частотой):
   - «кто где вообще» — медленный (~2 с), вся галактика → `net/presence.ts` (ЕСТЬ). Для
     списка, маркеров на галакт-карте, карточки в чате.
   - «где именно летит рядом» — быстрый (~12–15 Гц), только своя система → `net/pose.ts`
     (НОВЫЙ). Для рендера чужого борта в мире + точки на радаре.
   Медленный гейтит быстрый: подписываешься на быстрый канал только для тех, кто в твоей системе.

3. **Лазер → быстрый снаряд, ЕДИНО для всех целей** (без ветки «по игроку иначе» — дух
   CLAUDE.md). Скорость **~25–30 км/с** (подобрать замером в `scratch/`), не время. Дальность
   лазера остаётся 1.2–3.2 км. **Заметание коллизии по шагу** обязательно (raycast вдоль
   отрезка — иначе протыкает цель). Сейчас лазер — истинный hitscan (`castLaser` в
   `combat/weapons.ts`, урон в том же кадре); это ИЗМЕНЕНИЕ домена: лёгкая сущность рядом с
   `MissileEntity`.
   - **Последствие:** боты перестанут попадать без упреждения (сейчас не упреждают — луч был
     мгновенный; есть тест «упреждение промахивается мгновенным лучом»). Прицеливание
     `ai/autofight.ts` учит упреждать; тест пересмотреть под замер.

4. **Связь с игроком — кликом по портрету** (в списке «В СЕТИ» и в карточке), не только
   отдельной кнопкой вызова.

## Этап 1 — Встреча (видеть друг друга в одной системе)

1. **`net/pose.ts` — быстрый канал позы (транспорт).** RTDB `poses/{systemIndex}/{uid}` =
   {абсолютные pos, quat, vel}, ~12–15 Гц, `onDisconnect` чистит. `publishPose` (свою),
   `subscribePoses(systemIndex)` (чужие в этой системе). При прыжке — снять старый узел,
   переподписаться. Чистый транспорт, без интерполяции.
2. **`RemoteController` + рендер чужого борта.** Ключевое решение реализации: удалённый игрок —
   это **`ShipEntity` в `world.ships`**, помеченный как сетевой; так БОЙ единообразен (луч-снаряд
   бьёт любой `ShipEntity`, без if-remote). Его состояние не интегрируется физикой, а **ставится
   интерполяцией** из буфера снапшотов (app-слой перезаписывает state, шаг ИИ/физики его
   пропускает). Меш — существующий `render/scene/Ships.tsx`. Дисконнект → грейс-таймер → деспавн.
3. **Маркеры.** Точка на радаре в системе (по позе) + маркер на `GalaxyMap`/`SystemMap`
   (по presence.systemIndex; данные уже есть).
4. **Клик-по-портрету → вызов** (см. решение 4).

## Этап 2 — Кооп-бой

5. **Лазер-снаряд в домене** (решение 3): новая лёгкая сущность + заметание коллизии + правка
   `fireLasers`. Замер скорости в `scratch/`. Правка ИИ-упреждения и теста.
6. **Синхрон снарядов и урона.** Стреляешь → снаряд летит у всех (релей выстрела). Попадание в
   ТЕБЯ рассудит твой клиент (авторитет над своим HP) по своей истинной позе. Смерть/урон/
   `Notice` — по сети.

## Отложено (двери открыты)

- Colyseus authoritative-сервер, истинный hitscan + lag compensation.
- Мульти-арендность (мир на компанию), офлайн-криостаз, статы-эффекты — из
  `PLAN_setevoy_komponent.md`, когда дойдём.
